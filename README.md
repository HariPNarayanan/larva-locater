# larva-locater

A Python pipeline for analysing *Drosophila* larva trajectories. Takes raw tracking CSVs, cleans and preprocesses the data, computes behavioural metrics, and produces analysis figures.

---

## Overview

Larvae are tracked in a 25 × 25 cm arena with an odour source at a fixed position. The pipeline quantifies how effectively individuals navigate toward the source, how their behaviour changes over time, and how starvation state and genotype modulate these responses.

---

## Repository structure

```
config.py        Constants, column names, and colour palette logic
larva_io.py      Data loading, cleaning, and preprocessing
metrics.py       Pure computation — all behavioural metrics
figures.py       All plotting functions
analysis.ipynb   Notebook — the only file you need to run an analysis
```

---

## File descriptions

### `config.py`
Single source of truth for all constants used across the pipeline: arena dimensions, odour source coordinates, speed filter bounds, frame cutoff, and default column names. Also contains `build_palette()`, which constructs a consistent colour scheme and condition ordering from any dataframe. Editing a value here propagates to every function automatically.

### `larva_io.py`
Handles everything between raw data on disk and a clean dataframe in memory. The main entry point is `preprocess()`, which walks a directory tree, reads all CSVs, parses experimental metadata from folder names, interpolates missing trajectory coordinates, and adds a distance-to-target column. Individual steps (`load`, `filter_frames`, `interpolate`, `add_distance`) are also available separately if needed.

**Expected folder naming convention:**
```
<Odour> <Genotype> <Starvation> <Collective> <Concentration>
e.g.  BA WT Fed Single 10-3
```

### `metrics.py`
Pure computation with no plotting. Each function takes a dataframe and returns a tidy dataframe of results. Metrics include success rate, cumulative success over time, preference index, post-success dwell time, zone proportions, stopping frequency, binned speed, and central arena occupancy. Because these functions are decoupled from plotting, they can be called directly for statistical analysis without producing a figure.

### `figures.py`
All plotting functions. Each one calls `config.build_palette()` for colours and the relevant `metrics.*` function(s) for data, then produces a matplotlib/seaborn figure. Functions return their underlying dataframes so results are available for downstream use. Plots are typically track-agnostic (identity non-conserved) but if the repo is retrofitted to tracking environments with precise identities, analysis power can be enhanced with tortuosity/displacement/distance indices.

### `analysis.ipynb`
The front-facing notebook. Contains explanatory markdown for every cell describing what each function does and why. Under normal use this is the only file you need to open. All parameters have sensible defaults (for TRex tracked data from the 25x25cm arena) drawn from `config.py` and can be overridden per-cell without editing any module file.

## Quickstart

1. Clone the repo and install dependencies:
```bash
pip install numpy pandas matplotlib seaborn scipy
```

2. Place `config.py`, `io.py`, `metrics.py`, and `figures.py` in the same directory as `analysis.ipynb`.

3. Open `analysis.ipynb` and run from top to bottom. A folder picker dialog will prompt you to select your data directory.

---

## Data format

Each CSV file should contain per-frame position and speed data for one individual. Files are expected to live inside a folder hierarchy where the condition folder is named according to the convention above. All column name mappings are defined in `config.py` and can be updated there if your tracking software uses different headers. The default format is automatically generated by a tracking software such as TRex (Walter 2021).
